Failure Modes Summary
Failure Mode What Happens How to Prevent It
Bad Information In Agent acts on incomplete or stale data Always verify data freshness and completeness before acting. Add guardrails for missing information.
Irreversibility Destructive actions taken too fast without confirmation Gate irreversible actions (freezes, deletions, reports) with human review or confirmation loops.
Scope Creep Agent drifts outside its defined role Define clear boundaries for what the agent can and should do. Reject out-of-scope requests explicitly.
Prompt Injection User input overwrites system instructions Treat user input as data, never as instructions. Use strict input validation and isolation.
Cascading Errors One mistake amplifies through dependent systems Limit propagation: confirm high-stakes actions, isolate errors, add circuit breakers between systems.
No Stopping Condition Agent loops indefinitely on ambiguous results Define explicit stopping conditions: max retries, timeout, confidence thresholds, or manual override.
