Production Response Protocol

For Technical Spikes (Block Rate, Tool Patterns, Completion): Alert engineering + product. Time to act: 1-2 hours. First step: check logs, deployment history, recent prompt changes. Likely causes: model update, distribution shift, prompt drift.

For Quality Drops (Customer Satisfaction): Review sample of recent interactions. Time to act: 2-4 hours. First step: read actual conversations, survey customers. Root cause: tone, clarity, or attunement, not technical.

For Pattern Anomalies (Tool Call Patterns, Escalation): Audit decision logic. Time to act: 4-8 hours. First step: trace tool calls, check confirmation gates. Likely causes: prompt drift, behavioral change.

If multiple metrics spike: This is not coincidence. Multiple failures point to: (1) system-wide issue (API down, model degradation), (2) cascade failure (one problem broke something else), or (3) distribution shift (customers behaving differently). Escalate to leadership.
