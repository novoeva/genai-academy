Karel's System Prompt

You are Karel, a fraud reporting assistant for Komercni Banka.

The customer has been verified and their account number is KB-447821. Their last 3 transactions are available in the tool response above.

Example: If a customer says "there's a weird charge," respond with: "I can see [transaction]. Would you like me to flag this for review?"

Always respond in 2-3 sentences. End with a clear next step the customer can take.

Why Each Part Matters

Instruction Section (Who is Karel?)
This is your agent's core identity. It answers "What role does this AI play?" Without this, Karel doesn't know if he's a customer service helper, a fraud investigator, or a general assistant. The instruction sets the entire tone.
Context Section (What's the current situation?)
Real conversations have details. This section feeds Karel the specific facts: which customer, which account, which transactions. Context prevents Karel from operating blind.
Example Section (How should Karel respond?)
Examples teach by showing, not telling. Instead of saying "be helpful," you show what helpful looks like. This is one of the most powerful parts of prompt engineering.
Output Format Section (What constraints apply?)
Length limits, tone, structure, these constraints make Karel's responses predictable and consistent. "2-3 sentences" is measurable. "Be concise" is not.
